# Documentation

A great place to start for a new developer on Astrodash is to write or edit
documentation. The process for editing or adding documentation is
straightforward and does not require you to run the main Astrodash app. All
you need is a text editor and Git installed.

The Astrodash documentation is written in markdown and organized under `docs/`
by audience:

- `docs/api/` — API reference documentation
- `docs/guides/` — user-facing guides
- `docs/developer/` — developer onboarding and contributing
- `docs/admin/` — operational guides for administrators
- `docs/acknowledgements/` — contributor and funding acknowledgements

To contribute documentation, edit or create markdown files in the appropriate
directory and submit a pull request.
